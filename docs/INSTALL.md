# Конспект: настройка окружения Solana и Anchor

## 1. Что нужно установить

Для разработки программ на Solana обычно нужны три слоя инструментов: Rust, Solana CLI и Anchor CLI. Для удобства в современном стеке часто добавляют Node.js, Yarn и Docker, если используются TypeScript-тесты или verifiable builds/верификация сборок.

Если вы работаете на Windows, сначала установите WSL, а уже внутри Ubuntu/WSL ставьте остальные компоненты. На macOS и Linux можно идти сразу по шагам ниже.

## 2. Системные зависимости

Перед установкой убедитесь, что в ОС есть базовые библиотеки и утилиты для сборки нативных компонентов. На Linux/WSL обычно нужны `libssl-dev`, `libudev-dev`, `pkg-config`, `zlib1g-dev`, `llvm`, `clang`, `cmake`, `make`, `libprotobuf-dev`, `protobuf-compiler`, `libclang-dev`.

Пример для Ubuntu:

```bash
sudo apt-get update
sudo apt-get install -y libssl-dev libudev-dev pkg-config zlib1g-dev llvm clang cmake make libprotobuf-dev protobuf-compiler libclang-dev
```

Для macOS обычно достаточно установить Rust и затем перейти к Solana CLI и Anchor, потому что большинство зависимостей подтянется через стандартные инструменты. Для Windows используйте WSL и уже в нём выполняйте Linux-инструкции.

## 3. Быстрая установка одной командой

Официальная документация Solana предлагает быстрый сценарий: одна команда ставит Rust, Solana CLI и Anchor Framework сразу.

```bash
curl --proto '=https' --tlsv1.2 -sSfL https://solana-install.solana.workers.dev | bash
```

После выполнения проверьте версии:

```bash
rustc --version && solana --version && anchor --version && surfpool --version && node --version && yarn --version
```

Этот способ удобен для учебных групп, потому что уменьшает число ручных ошибок при установке.

## 4. Установка по компонентам

Если быстрый сценарий не подходит, ставьте всё вручную. Такой способ полезен, когда нужно зафиксировать конкретные версии, повторить окружение на CI или объяснить студентам, что делает каждый инструмент.

### 4.1 Установка Rust

Рекомендуемый способ — через `rustup`:

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source "$HOME/.cargo/env"
rustc --version
cargo --version
```

Rust нужен потому, что Solana-программы пишутся на Rust, а Anchor строится поверх Rust-макросов и toolchain.

### 4.2 Установка Solana CLI

Официальный инсталлятор позволяет выбрать канал `stable`, `beta`, `edge` или конкретный релизный тег.

```bash
sh -c "$(curl -sSfL https://release.solana.com/stable/install)"
solana --version
```

Если нужно зафиксировать версию, замените `stable` на нужный тег, например `v2.0.3`.

Для обновления Solana CLI в дальнейшем используйте тот же подход с нужным каналом или релизом. Официальная документация также указывает, что актуальные релизы доступны в репозитории Agave, который является современным validator client от Anza.

### 4.3 Установка Anchor через AVM

**AVM — рекомендованный способ** установки Anchor, потому что он позволяет держать несколько версий и переключаться между ними.

```bash
cargo install --git https://github.com/coral-xyz/anchor avm --force
avm --version
avm install latest
avm use latest
anchor --version
```

Если курс или проект требует конкретную версию, ставьте её явно:

```bash
avm install 0.32.0
avm use 0.32.0
anchor --version
```

Это особенно важно, потому что Anchor релизы завязаны на совместимость с конкретными версиями Solana.

### 4.4 Установка Anchor без AVM

Прямую установку из GitHub тоже можно использовать, но для учебной среды это менее удобно. Минус такого пути в том, что переключение между версиями и сопровождение нескольких проектов становятся ручными.

Если нужен стабильный учебный конвейер, AVM почти всегда лучше.

## 5. Совместимость Anchor и Solana

На практике студенты чаще всего спотыкаются не на самой установке, а на несовместимости версий. Поэтому в учебном проекте лучше сразу зафиксировать пару Anchor/Solana в инструкции к репозиторию.

Ориентир по официальным release notes:


| Anchor | Рекомендуемая Solana | Примечание |
| :-- | --: | :-- |
| 0.29.0 | 1.16.0+ | Solana 1.14 больше не поддерживается.  |
| 0.30.0 | 1.18.8 | Рекомендуемая версия указана в релиз-нотах.  |
| 0.31.0 | 2.1.0 | Переход на Agave-нейминг бинарников.  |
| 0.32.0 | 2.3.0 | В релиз-нотах указана эта рекомендованная версия.  |

Практическое правило простое: если проект уже создан, сначала проверьте `Anchor.toml`, `Cargo.toml`, release notes и только потом ставьте версию через AVM.

## 6. Настройка кошелька и сети

После установки инструментов настройте Solana CLI конфигурацию и кошелёк. Официальная документация рекомендует проверить текущую конфигурацию, затем переключиться на нужный кластер и создать keypair.

```bash
solana config get
solana config set --url devnet
solana-keygen new
solana address
solana balance
```

Если нужен локальный стенд для экспериментов, запускайте local validator:

```bash
solana-test-validator
```

При работе с devnet можно запросить airdrop для тестов:

```bash
solana airdrop 2
solana balance
```


## 7. Создание проекта

После установки создайте новый Anchor-проект:

```bash
anchor init my-project
cd my-project
```

По умолчанию Anchor создаёт модульную структуру проекта и использует тестовый шаблон LiteSVM. При необходимости можно выбрать другой шаблон тестов через `--test-template`.

Проверить структуру и базовую сборку можно так:

```bash
anchor build
```

Скомпилированный `.so` файл появится в `target/deploy`.

## 8. Запуск тестов

Для проверки проекта используйте:

```bash
anchor test
```

Эта команда собирает проект, разворачивает его в тестовой среде и запускает тесты. Если выбран `localnet`, Anchor сам поднимает локальный валидатор, деплоит программу, выполняет тесты и завершает процесс.

Если в проекте используются TypeScript-тесты, убедитесь, что установлены Node.js и Yarn. Официальная документация Anchor отмечает, что они нужны для инициализации проекта и шаблонов на базе TypeScript.

## 9. Проверка готовности

Окружение считается готовым, если выполняются все проверки ниже:

```bash
rustc --version
cargo --version
solana --version
anchor --version
solana config get
solana address
anchor build
anchor test
```

Дополнительно полезно проверить, что `solana --version` и `anchor --version` совпадают с версией, которую требует ваш курс или шаблон проекта. Это снижает риск ошибок на следующем шаге — написании и деплое программы.
